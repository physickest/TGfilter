# TGfilter
filter for TG   【currently not with Vision model used. Looking forward to a breakthrough!】

Project TG-Filter-WatchdogStatus: Final Release / Post-Mortem AnalysisDate: 2026-02-06Focus: OOD Generalization, Zero-Day Leak Detection, Headless Deployment1. AbstractAddressing the conflict between "high-value information acquisition" and "low-value noise/spoiler avoidance" in Telegram channels, this experiment aimed to construct an automated interception and forwarding system. The core objective was to solve the specific challenge of precise identification and redirection of "Zero-day Spoiling Images" (unseen leak content) without human intervention.2. Problem StatementHuman Bottleneck: Users rely on "manual checks" to distinguish image content, leading to low information retrieval efficiency and an inability to decouple from channel monitoring.Model Boundaries (The Knowledge Cutoff): Existing pre-trained models (e.g., CLIP, WD14) suffer from Knowledge Cutoff. When facing Out-of-Distribution (OOD) data—such as newly released game characters—they exhibit severe classification deviation.Environmental Constraints: The system must be deployed on headless compute nodes (e.g., GitHub Actions), requiring a lightweight and non-GUI architecture.3. In the model's weight space, these unknown features were forcibly projected onto the nearest known clusters (e.g., incorrectly classifying them as generic game assets or wrong characters). This "hallucination" confirms the inadequacy of static weights when processing real-time streaming data.4. Technical Discussion4.1 Cognitive Upgrade: From Classification to ReasoningThe experiment demonstrates that AI should not be treated as an "Encyclopedia," but rather as a "Feature Extractor."Classification Mindset (Failed):$$f(x) \to \text{Label}$$When $x$ is a new class (unseen data), the output is inevitably erroneous.Reasoning/Contrastive Mindset (Effective):Utilizing Zero-shot Generalization. We calculate the Cosine Similarity between the image vector $V_{img}$ and the target concept vector $V_{target}$ (e.g., "Anime Leak") in the semantic space.4.2 The OOD (Out-of-Distribution) ChallengeKey Insight: In research, when SOTA models collapse on large-scale OOD data, it highlights the necessity of Engineering Architecture. The solution lies in establishing a "Dynamic Feedback Loop" rather than relying solely on pre-trained knowledge.5. ConclusionReject Mediocre Automation: Simple forwarding scripts cannot solve semantic-level spoiler issues.Embrace Uncertainty: The AI system must possess the capability to handle "I don't know" states rather than forcing a wrong answer.Robustness First: The optimal filtering system is HITL (Human-in-the-loop). If the user flags one image, the system should automatically shield against a subsequent batch of visually similar data.👨‍💻 Tech Lead's Note"The failure of this experiment is a necessary step toward a PhD. Instead, use engineering architecture to constrain that 10% uncertainty."
