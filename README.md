AbstractAddressing the conflict between "high-value information acquisition" and "low-value noise/spoiler avoidance" in Telegram channels, this experiment aimed to construct an automated interception and forwarding system. The core objective was to solve the specific challenge of precise identification and redirection of "Zero-day Spoiling Images" (unseen leak content) without human intervention.

2. Problem Statement
Human Bottleneck: Users rely on "manual checks" to distinguish image content, leading to low information retrieval efficiency and an inability to decouple from channel monitoring.
Model Boundaries (The Knowledge Cutoff): Existing pre-trained models (e.g., CLIP, WD14) suffer from Knowledge Cutoff. When facing Out-of-Distribution (OOD) data—such as newly released game characters—they exhibit severe classification deviation.
Environmental Constraints: The system must be deployed on headless compute nodes (e.g., GitHub Actions), requiring a lightweight and non-GUI architecture.

In actual testing, General Purpose LLMs/VLMs failed to correctly identify all four images.
Root Cause:
These images represented "Zero-day" leak content released within the same week as the experiment. In the model's weight space, these unknown features were forcibly projected onto the nearest known clusters (e.g., incorrectly classifying them as generic game assets or wrong characters). This "hallucination" confirms the inadequacy of static weights when processing real-time streaming data.

4. Technical Discussion

4.1 Cognitive Upgrade: From Classification to ReasoningThe experiment demonstrates that AI should not be treated as an "Encyclopedia," but rather as a "Feature Extractor."

Classification Mindset (Failed):$$f(x) \to \text{Label}$$When $x$ is a new class (unseen data), the output is inevitably erroneous.
Reasoning/Contrastive Mindset (Effective):Utilizing Zero-shot Generalization. We calculate the Cosine Similarity between the image vector $V_{img}$ and the target concept vector $V_{target}$ (e.g., "Anime Leak") in the semantic space.

4.2 The OOD (Out-of-Distribution) Challenge
Key Insight: In research, when SOTA models collapse on large-scale OOD data, it highlights the necessity of Engineering Architecture. The solution lies in establishing a "Dynamic Feedback Loop" rather than relying solely on pre-trained knowledge.

5. Proposed Architecture
# 🏗️ Proposed Architecture

> A one-shot visual interception pipeline combining semantic perception, local vector memory, and agentic metadata fallback.

---

## Module Overview

| Module | Tech Stack | Logic Description |
|:---:|:---:|---|
| 🔍 **Perception** | `DINOv2` / `OpenCLIP` | Extracts deep semantic vectors. Does not require specific naming or labeling — only precise feature representation. |
| 🧠 **Memory** | `Vector Cache (Local)` | Stores feature vectors of 1–2 positive samples manually confirmed by the user *(one-shot learning)*. |
| ⚖️ **Decision** | `Cosine Similarity` | Triggers interception when the distance between new image **V**_new and cached **V**_ref is below threshold **θ**. |
| 🤖 **Agentic** | `Metadata` + `OCR` | Acts as a redundancy backup by combining Regex matching on message captions and metadata. |

## Module Details

### 🔍 Perception
- **Stack:** DINOv2, OpenCLIP
- Encodes input images into high-dimensional feature vectors.
- Model-agnostic — no task-specific fine-tuning or class labels required.

### 🧠 Memory
- **Stack:** Local Vector Cache
- Stores reference embeddings (**V**_ref) from 1–2 manually confirmed positive samples.
- Enables **one-shot learning** without retraining.

### ⚖️ Decision
- **Stack:** Cosine Similarity
- Computes similarity score between **V**_new and **V**_ref.
- Intercepts when: `cos_sim(V_new, V_ref) ≥ θ`

### 🤖 Agentic
- **Stack:** Metadata, OCR, Regex
- Secondary fallback layer when visual similarity alone is inconclusive.
- Scans message captions and file metadata using pattern matching.


7. Conclusion
Reject Mediocre Automation: Simple forwarding scripts cannot solve semantic-level spoiler issues.

Embrace Uncertainty: The AI system must possess the capability to handle "I don't know" states rather than forcing a wrong answer.

Robustness First: The optimal filtering system is HITL (Human-in-the-loop). If the user flags one image, the system should automatically shield against a subsequent batch of visually similar data.

Do not revert to manual operations because of a 10% model failure rate. Instead, use engineering architecture to constrain that 10% uncertainty."
